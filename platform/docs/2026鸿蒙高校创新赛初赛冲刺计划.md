## 1. 最终参赛定位

GuardianHub 不是四个松散工具，而是一个面向高校学生的“操作前数字安全闸门”：

- 分享图片前：Privacy Sentinel 隐私哨兵
- 提交项目代码前：Code Guardian 代码卫士
- 点击链接前：Link Guard 链接卫士
- 上传作业或报名材料前：Doc Shield 提交护盾

四个模块统一输出：

- `high / medium / low` 风险等级
- 0-100 安全评分
- 风险证据
- 建议操作
- 是否建议继续分享、提交、点击或上传

初赛核心叙事：

> GuardianHub 将分散的校园数字风险检查前置到用户关键操作之前，并借助 HarmonyOS 的系统文件选择、图片处理、分享服务和本地数据能力，形成“检测—解释—处置—继续操作”的统一安全闭环。

一句话创意描述候选：

> 分享、提交和点击之前，先过一道智能安全闸门

## 3. 产品深度分配

| 模块 | 初赛深度 | 必须演示的闭环 |
| --- | --- | --- |
| Privacy Sentinel | 主模块、完整闭环 | 系统选图 → OCR/二维码/隐私识别 → 风险框 → 黑条/模糊/马赛克 → 保存并分享 |

原则：四个模块都可用，但演示时间主要留给 Privacy Sentinel 和 Code Guardian。

## 5. Code Guardian ZIP 项目扫描改造

### 5.1 结论

支持 ZIP 项目包值得做，能够把 Code Guardian 从“代码片段检查器”升级为“学生项目提交前安全检查器”，更符合课程项目、竞赛作品和开源仓库场景。

### 5.2 接口策略

继续使用现有：

`POST /api/code/analyze`

根据上传文件后缀分支：

- 普通代码文件：保持当前逻辑。
- `.zip`：进入项目扫描逻辑。

这样 Web 和 HarmonyOS 只需要接入一个接口。

### 5.3 后端文件改造

- [x] 新增 `backend/modules/code_guardian/archive_scanner.py`。
- [x] 修改 `backend/main.py`，删除当前 ZIP 拒绝逻辑。
- [x] 扩展 `backend/schemas/models.py`。
- [x] 扩展 `backend/config.py` 和 `.env.example`。
- [ ] 修改 `backend/modules/code_guardian/report_generator.py` 支持项目聚合。

建议响应字段：

scanMode: single | project

### 5.5 ZIP 安全边界

必须满足：

- [x] 只在内存中读取 ZIP 条目，不把用户目录结构直接解压到文件系统。
- [x] 拒绝包含 `..`、绝对路径、盘符路径的条目，防止 Zip Slip。
- [x] 拒绝损坏或加密 ZIP。
- [x] 跳过符号链接和非普通文件。
- [x] 压缩包大小上限：10 MB。
- [x] 解压后总大小上限：50 MB。
- [x] 最大条目数：300。
- [x] 单个代码文件上限：1 MB。
- [x] 最大压缩比：100:1。
- [x] 跳过二进制文件和无法可靠解码的文件。
- [x] 不执行、编译或导入用户代码。

新增配置建议：

```text
GUARDIANHUB_MAX_CODE_ARCHIVE_BYTES=10485760
GUARDIANHUB_MAX_CODE_UNCOMPRESSED_BYTES=52428800
GUARDIANHUB_MAX_CODE_ARCHIVE_FILES=300
GUARDIANHUB_MAX_CODE_FILE_BYTES=1048576
GUARDIANHUB_MAX_CODE_COMPRESSION_RATIO=100
```

### 5.6 扫描与评分

- 每个文件复用现有 `analyze_code()`。
- 风险项增加 `filePath`。
- 项目总分不做简单平均，按最高风险和风险密度扣分。
- `high` 风险文件至少出现一个时，项目不能显示“可以直接提交”。
- 报告展示：
  - 扫描文件数
  - 跳过文件数
  - 语言分布
  - 高/中/低风险数量
  - 风险最高的 5 个文件
  - 文件路径、行号、代码片段、原因、建议

联网增强原则：

- 不把整个 ZIP 发给 DeepSeek。
- 本地扫描全部文件。
- 用户主动选择联网增强后，只发送去重后的风险摘要和最多 20 个必要代码片段。
- UI 明确告知哪些内容可能被发送。

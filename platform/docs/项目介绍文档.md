# GuardianHub：面向高校场景的 AI 数字安全防护平台

## 项目定位

GuardianHub 是面向高校场景的 AI 数字安全防护平台，由原单一 Privacy Sentinel 隐私哨兵升级而来。Privacy Sentinel 保留为平台中的隐私哨兵模块，平台整体围绕“分享前、点击前、转账前、提交前”四个关键节点，为学生在社交分享、聊天沟通、链接访问、材料提交等高频场景提供可解释的风险提示和处理建议。

当前版本不接入付费大模型 API，优先使用本地规则和 mock 数据保证比赛演示、课堂展示和本地运行的稳定性。

## 背景痛点

高校学生每天都会处理截图分享、奖助学金通知、社团报名、课程作业、实习招聘、二手交易和校园群聊消息。很多风险不是发生在复杂系统里，而是发生在一次点击、一次转发或一次材料提交前：

- 图片中无意暴露手机号、地址、二维码、订单号或聊天昵称。
- 群聊消息诱导填写银行卡、验证码、身份证等敏感信息。
- 短链接或仿冒域名伪装成学校通知、奖学金补贴或账号验证入口。
- 作业和报名材料提交前命名不规范、附件遗漏或包含无关隐私字段。

GuardianHub 将这些分散风险整合为一个安全中心，帮助学生养成提交前检查的习惯。

## 平台模块

### Privacy Sentinel 隐私哨兵

保留原有核心能力：图片上传、隐私检测、风险分级、敏感区域标注、一键打码、安全预览和历史记录。适合截图、快递单、订单页、聊天记录等图片分享场景。

### Scam Radar 反诈雷达

支持输入聊天文本，当前基于规则识别验证码、转账垫付、中奖补贴、紧迫话术、冒充客服或老师、刷单兼职、索要敏感资料等风险。输出统一风险等级、命中原因和建议操作。

### Link Guard 链接卫士

支持输入 URL，检查是否 HTTPS、是否短链接、是否包含可疑关键词、域名是否异常。二维码解析后的链接后续也可以复用同一检测接口。

### Doc Shield 提交护盾

当前为静态 / mock 页面，展示文件命名检查、隐私检查、材料清单检查三类能力。后续可扩展为真实文件解析、格式校验和材料模板核验。

## 技术路线

- 前端：React + Vite + TypeScript，首页已重构为 GuardianHub 安全中心。
- 后端：FastAPI 提供 `/api/detect`、`/api/mask`、`/api/history` 以及新增安全模块接口。
- 图像处理：Pillow 完成真实打码、模糊和马赛克处理。
- 风险策略：所有模块统一使用 `high`、`medium`、`low` 风险等级。
- 演示策略：规则和 mock 数据优先，确保本地稳定运行。

## 后续扩展

复赛或产品化阶段可逐步接入 OCR、二维码解析、人脸检测、文件解析、学校通知源校验和更强的语义识别模型。HarmonyOS / ArkTS 原型目录已保留，可作为后续多端迁移基础。

# GuardianHub 项目介绍

GuardianHub 是面向高校场景的 AI 数字安全防护平台。它从原先的 Privacy Sentinel 隐私工具升级为安全中心，覆盖“图片分享前、代码提交前、链接点击前、材料提交前”四类高频风险。

## 核心目标

GuardianHub 不追求堆叠无关功能，而是把四个安全工具统一成一个可解释、可演示、可扩展的平台：

- 统一风险等级：`high` / `medium` / `low`
- 统一安全评分：0-100
- 统一报告结构：风险等级、检测摘要、风险证据、建议操作
- 统一视觉风格：卡片式、科技感、安全感、简洁清晰

## 四个模块

### Privacy Sentinel 隐私哨兵

用于图片分享前隐私检测。系统会在原图上标出手机号、证件号、银行卡号、邮箱、地址、二维码和订单号等风险区域，生成隐私检测报告，并支持黑条、模糊、马赛克三种处理方式。人脸检测需要单独配置本地模型，默认关闭。

### Code Guardian 代码卫士

用于学生项目、课程实验、竞赛作品和开源代码提交前的轻量级安全检测。用户可以粘贴代码、上传单个文件，或上传项目 ZIP。系统在内存中只读扫描源码，不执行项目、不安装依赖，自动忽略依赖与构建目录，并防护路径穿越、符号链接和压缩炸弹。报告按文件展示硬编码密钥、SQL 注入、命令执行、路径穿越、弱加密、敏感日志、危险配置和 XSS 风险，给出风险文件优先级与是否建议提交。

### Link Guard 链接卫士

用于链接与二维码安全体检。用户输入 URL、短链接、二维码解析内容或链接来源说明后，系统从协议、域名、参数、关键词、随机 token、仿冒域名和来源场景等角度生成链接安全体检报告，并给出是否建议打开。

### Doc Shield 提交护盾

用于作业、报名材料、报告等提交前检查。流程为输入提交要求、上传材料、生成提交检查报告。报告包含材料完整性、格式规范、隐私风险、提交建议和安全评分。

## 技术实现

- HarmonyOS 客户端：HarmonyOS 6、ArkTS、Stage 模型，接入系统图片/文件 Picker 与四模块 API
- PC Web：React 18 + Vite + TypeScript，承担完整能力演示和接口验收
- 检测服务：FastAPI + Pydantic v2，SQLite 保存检测历史
- 本地能力：RapidOCR、OpenCV、Pillow、正则与规则引擎
- 可选增强：DeepSeek 与 Qwen VL，默认关闭，仅在用户主动选择联网模式并配置服务后调用

## 可扩展方向

当前版本已经接入本地 OCR、二维码定位和代码项目 ZIP 扫描，并完成统一评分、SQLite 历史、上传限制和自动化测试。HarmonyOS 工程可编译并生成调试 HAP，四模块 Picker 与接口、ArkData 本地历史、处理图预览/系统保存和 Share Kit 已经接入。未来可继续加入依赖漏洞库、校内通知可信源校验和端侧模型。

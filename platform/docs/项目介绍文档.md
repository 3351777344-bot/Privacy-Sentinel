# GuardianHub 项目介绍

GuardianHub 是一个统一的 AI 数字安全防护平台，面向高校学生和普通用户的日常数字安全场景。平台从原先的 Privacy Sentinel 隐私工具升级为安全中心，覆盖“图片分享前、聊天转账前、链接点击前、材料提交前”四类高频风险。

## 核心目标

GuardianHub 不追求堆叠大量无关功能，而是把已有四个安全工具统一成一个可解释、可演示、可扩展的平台：

- 统一风险等级：`high` / `medium` / `low`
- 统一安全评分：0-100
- 统一报告结构：风险等级、检测摘要、风险证据、建议操作
- 统一视觉风格：卡片式、科技感、安全感、简洁清晰

## 安全中心

平台首页是“安全中心”，展示 GuardianHub 介绍、今日安全评分、四个模块卡片和最近安全检测历史。每个模块卡片都显示最近检测状态、风险等级和安全评分，让用户能先看整体态势，再进入具体工具处理风险。

## 四个模块

### 隐私哨兵

用于图片分享前隐私检测。系统会在原图上标出手机号、地址、二维码、订单号、昵称等风险区域，生成隐私检测报告，并支持三种处理策略：

- 只处理高风险
- 处理全部
- 自定义选择

处理完成后，页面展示原图与处理后图片对比，方便用户复核后再分享。

### 反诈雷达

用于聊天文本诈骗风险识别。系统不只输出风险等级，还输出命中的风险证据和防骗建议。当前规则覆盖：

- 诱导转账
- 高额回报
- 催促决策
- 脱离平台
- 隐瞒他人
- 可疑链接
- 索要验证码或敏感资料
- 冒充老师、客服、银行、公安等身份

### 链接卫士

用于 URL 访问前体检。系统检查：

- HTTPS
- 短链接
- 可疑关键词
- 异常域名
- URL 过长
- 随机字符路径
- 可疑参数

输出链接体检报告、风险原因和访问建议。

### 提交护盾

用于作业、报名材料、报告等提交前检查。流程为：

1. 输入提交要求
2. 上传材料
3. 生成提交检查报告

报告包含材料完整性、格式规范、隐私风险、提交建议和安全评分。当前不强依赖复杂 OCR，图片和压缩包会先做文件名、后缀和上传状态检查。

## 技术实现

- 前端：React + Vite + TypeScript
- 后端：FastAPI
- 图片处理：Pillow，本地打码、模糊、马赛克
- 风险识别：本地规则和 mock 检测，不接入付费大模型 API
- 历史记录：后端本地 JSON 文件保存最近检测记录

## 可扩展方向

后续可在不改变平台结构的前提下接入 OCR、二维码解析、校内通知可信源校验、诈骗语义识别或端侧模型，但当前版本优先保证本地演示稳定和风险报告可解释。

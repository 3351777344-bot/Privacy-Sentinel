# GuardianHub 项目介绍

GuardianHub 是面向高校场景的 AI 数字安全防护平台。它从原先的 Privacy Sentinel 隐私工具升级为安全中心，覆盖“图片分享前、代码提交前、链接点击前、材料提交前”四类高频风险。

## 核心目标

GuardianHub 不追求堆叠无关功能，而是把四个安全工具统一成一个可解释、可演示、可扩展的平台：

- 统一风险等级：`high` / `medium` / `low`
- 统一安全评分：0-100
- 统一报告结构：风险等级、检测摘要、风险证据、建议操作
- 统一视觉风格：卡片式、科技感、安全感、简洁清晰

## 四个模块

### Privacy Sentinel 隐私哨兵

用于图片分享前隐私检测。系统会在原图上标出手机号、地址、二维码、订单号、头像昵称等风险区域，生成隐私检测报告，并支持黑条、模糊、马赛克三种处理方式。

### Code Guardian 代码卫士

用于学生项目、课程实验、竞赛作品和开源代码提交前的轻量级安全检测。用户可以粘贴代码或上传单个代码文件，系统基于本地规则检测硬编码密钥、SQL 注入、命令执行、路径穿越、弱加密、敏感日志、危险配置和 XSS 风险，并输出修改建议与是否建议提交。

### Link Guard 链接卫士

用于链接与二维码安全体检。用户输入 URL、短链接、二维码解析内容或链接来源说明后，系统从协议、域名、参数、关键词、随机 token、仿冒域名和来源场景等角度生成链接安全体检报告，并给出是否建议打开。

### Doc Shield 提交护盾

用于作业、报名材料、报告等提交前检查。流程为输入提交要求、上传材料、生成提交检查报告。报告包含材料完整性、格式规范、隐私风险、提交建议和安全评分。

## 技术实现

- 前端：React + Vite + TypeScript
- 后端：FastAPI
- 图片处理：Pillow，本地打码、模糊、马赛克
- 风险识别：本地规则、正则和关键词，不接入付费大模型 API
- 历史记录：后端本地 JSON 文件保存最近检测记录

## 可扩展方向

后续可在不改变平台结构的前提下接入 OCR、二维码解析、校内通知可信源校验、依赖漏洞库、代码项目级扫描或端侧模型。当前版本优先保证本地演示稳定和风险报告可解释。

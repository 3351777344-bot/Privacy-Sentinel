# GuardianHub 初赛 800 字介绍

GuardianHub：面向高校场景的 AI 数字安全防护平台，是由原 Privacy Sentinel 隐私哨兵升级而来的校园数字安全中心。平台面向高校学生日常生活中的图片分享、代码提交、链接访问和材料提交四类高频场景，把隐私泄露、代码安全缺陷、可疑链接和提交前疏漏等风险前置到关键操作之前，帮助学生在“分享前、提交前、点击前、上传前”完成快速自检。

平台保留 Privacy Sentinel 隐私哨兵作为核心模块：用户上传截图、快递单、订单页或聊天记录图片后，系统会自动生成隐私检测结果，在图片上标注手机号、地址、二维码、订单号、头像昵称等敏感区域，并按照 high、medium、low 进行风险分级。对于高风险信息，系统给出明确处理建议，并支持黑条、模糊、马赛克三种真实图像处理方式，用户可以在安全预览中确认处理效果，再决定是否分享。

在平台升级后，GuardianHub 新增 Code Guardian 代码卫士、Link Guard 链接卫士和 Doc Shield 提交护盾。Code Guardian 面向学生项目、课程实验、竞赛作品和开源代码，支持粘贴代码或上传单个代码文件，基于本地规则检测硬编码密钥、SQL 注入、命令执行、路径穿越、弱加密、敏感日志、危险配置和 XSS 风险，输出代码安全体检报告和修改建议。Link Guard 支持输入 URL、短链接、二维码解析内容和链接来源说明，从协议、域名、参数、关键词、随机 token、仿冒域名和来源场景生成链接安全体检报告，并判断是否建议打开。Doc Shield 面向作业、报名表、报告等材料提交前检查，展示材料完整性、格式规范、隐私风险和提交建议。

当前阶段项目采用 PC Web Demo 进行验证：前端使用 React + Vite + TypeScript，后端使用 FastAPI，图像处理使用 Pillow，本地文字识别使用 RapidOCR 与 ONNX Runtime，二维码定位使用 OpenCV。平台不接入付费大模型 API，统一使用风险等级、0-100 安全评分、命中原因和建议操作构建可解释闭环，并通过上传限制、自动清理、SQLite 历史和自动化测试提升演示可靠性。后续将逐步加入代码项目级扫描、依赖漏洞库、校内通知可信源校验和 HarmonyOS / ArkTS 多端体验。

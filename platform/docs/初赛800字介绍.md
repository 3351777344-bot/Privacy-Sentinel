# GuardianHub 初赛 800 字介绍

GuardianHub：面向高校场景的 AI 数字安全防护平台，是由原 Privacy Sentinel 隐私哨兵升级而来的校园数字安全中心。平台面向高校学生日常生活中的图片分享、聊天沟通、链接访问和材料提交四类高频场景，把隐私泄露、诈骗诱导、可疑链接和提交前疏漏等风险前置到关键操作之前，帮助学生在“分享前、转账前、点击前、提交前”完成快速自检。

平台保留 Privacy Sentinel 隐私哨兵作为核心模块：用户上传截图、快递单、订单页或聊天记录图片后，系统会自动生成隐私检测结果，在图片上标注手机号、地址、二维码、订单号、头像昵称等敏感区域，并按照 high、medium、low 进行风险分级。对于高风险信息，系统会给出明确处理建议，并支持黑条、模糊、马赛克三种真实图像处理方式，用户可以在安全预览中确认处理效果，再决定是否分享。

在平台升级后，GuardianHub 新增 Scam Radar 反诈雷达、Link Guard 链接卫士和 Doc Shield 提交护盾。Scam Radar 支持粘贴聊天文本，识别验证码索取、转账垫付、中奖补贴、冒充客服、刷单兼职等诈骗话术；Link Guard 支持输入 URL，检查 HTTPS、短链接、可疑关键词和异常域名，后续可复用到二维码解析链接；Doc Shield 面向作业、报名表、报告等材料提交前检查，展示文件命名、隐私字段和材料清单三类能力。

当前阶段项目采用 PC Web Demo 进行初赛验证：前端使用 React + Vite + TypeScript，后端使用 FastAPI，图像处理使用 Pillow，检测策略优先使用本地规则和 mock 数据，保证演示稳定可靠。GuardianHub 的创新点在于将分散的校园数字风险整合为一个安全中心，用统一风险等级、命中原因和建议操作构建可解释闭环。后续将逐步接入 OCR、二维码识别、人脸检测、文件解析和更完善的语义风险识别能力，并迁移到 HarmonyOS / ArkTS 多端体验。

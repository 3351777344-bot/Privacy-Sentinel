# GuardianHub 数字安全防护平台 · 5 分钟展示文稿

> 目标时长：≤ 5 分钟 ｜ 形式：PC Web Demo 现场演示 + 讲解
> 建议节奏：讲解与操作同步进行，演示前先启动前后端并预置好示例数据。

---

## 0:00–0:30 开场与定位

各位老师、同学好，我们的项目是 **GuardianHub 数字安全防护平台**。

它由隐私哨兵升级而来，聚焦一个核心痛点：高校学生在**分享图片、提交代码、点击链接、上传材料**时，往往在操作发生后才意识到风险。GuardianHub 的思路是把安全检查**前置**到这四个关键动作之前，做一个统一的「操作前安全中心」。

一句话概括：**本地优先、联网可选、可解释、可闭环。**

---

## 0:30–1:00 整体设计

平台围绕四个风险节点提供四个模块：

- **分享前** → Privacy Sentinel 隐私哨兵
- **提交前** → Code Guardian 代码卫士
- **点击前** → Link Guard 链接卫士
- **上传前** → Doc Shield 提交护盾

所有模块统一输出三样东西：`high / medium / low` **风险等级**、**0–100 安全评分**、**命中证据与处置建议**，让结果始终可解释、可操作。检测记录统一存入本地 SQLite，可回放、可按模块统计平均分。

**关键设计**：图片和代码检测都能在界面上选择「本地处理」或「联网增强」；链接与材料检查始终纯本地静态完成。默认走本地，只有用户主动选联网且已配置模型，才会调用外部 API。

## 1:00–3:30 功能演示

### ① 首页安全中心（约 10 秒）
打开 Demo 首页，展示四个模块入口、今日安全评分、平台介绍与最近检测记录。

### ② Privacy Sentinel 隐私哨兵（约 50 秒）
上传一张示例图片（快递单 / 聊天截图）。系统在原图上框出**手机号、地址、二维码、订单号、昵称**等敏感区域，并给出风险分级。

- 切换「本地处理 / 联网增强」：本地用 OCR + 正则 + 二维码定位；联网可用 Qwen VL 视觉增强，失败自动回退本地。
- 选择打码范围（高风险 / 全部 / 自定义），后端用 Pillow 对**原图真实处理**，支持黑条、模糊、马赛克，导出安全分享版本。

**强调**：这不是打个马赛克预览，而是生成真正脱敏后的新图片。

### ③ Code Guardian 代码卫士（约 40 秒）
粘贴一段含**硬编码 API Key、SQL 拼接、命令执行、敏感日志**的示例代码，点击检测。

- 本地规则即可给出命中行号、代码片段、风险原因、修改建议和「是否建议直接提交」。
- 切到联网增强，DeepSeek 做深度分析，并支持**一键修复**生成安全代码。

### ④ Link Guard 链接卫士（约 30 秒）
输入一个含 `login`、`redirect`、`token` 的 URL，来源选「短信」。系统从 HTTPS、可疑参数、关键词、随机 token、仿冒域名和来源场景生成体检报告，直接告诉你**是否建议打开**。

也可上传二维码图片，系统在**本机解析**出链接自动填入——**全程不主动访问目标地址，避免 SSRF 风险**。

### ⑤ Doc Shield 提交护盾（约 20 秒）
输入课程论文的提交要求并上传材料，系统从 PDF / DOCX / TXT 中提取内容，检查**材料完整性、格式规范、命名规则和隐私信息**，给出提交建议。

## 3:30–4:15 技术实现

- **前端**：React 18 + Vite + TypeScript；**后端**：FastAPI + Pydantic v2。
- **本地能力**：RapidOCR（ONNX Runtime）文字识别、OpenCV 二维码/人脸定位、正则与规则引擎、Pillow 图像打码。
- **可选增强**：DeepSeek（代码分析与修复）、Qwen VL（图片视觉检测），走 OpenAI 兼容接口，默认关闭。
- **存储与安全**：本地 SQLite 历史；上传做真实格式与像素校验、大小限制、24 小时自动清理；CORS 仅放行本地地址；上传文件与数据库不入 Git。
- 已配置 GitHub Actions，推送即跑后端 pytest 与前端 build。

---

## 4:15–4:45 创新点

1. **从单点工具到平台**：把分散的隐私、代码、链接、材料风险整合进一个统一安全中心。
2. **本地优先、联网可选**：默认本机可跑、可离线、可控；联网增强是加分项而非依赖项，失败自动回退。
3. **统一可解释闭环**：所有模块共用同一套「等级 + 评分 + 证据 + 建议」，结果人能看懂、能据此决策。

---

## 4:45–5:00 总结与规划

当前是可运行的 PC Web 验证版，四个模块已跑通完整闭环。后续计划：代码项目级扫描与依赖漏洞库、校内可信通知源校验，以及向 **HarmonyOS / ArkTS 多端**迁移，让安全检查真正嵌入手机、手表等日常入口。

**GuardianHub，让每一次分享、提交和点击之前，先过一道安全关。** 谢谢！

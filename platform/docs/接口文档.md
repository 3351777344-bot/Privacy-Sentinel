# GuardianHub 接口文档

后端默认地址：`http://127.0.0.1:8000`

所有模块统一使用 `high`、`medium`、`low` 风险等级。

## GET /api/health

检查后端服务是否正常运行。

```json
{
  "status": "ok",
  "message": "GuardianHub backend is running"
}
```

## POST /api/detect

Privacy Sentinel 图片隐私检测接口。

请求类型：`multipart/form-data`

字段：

- `file`：图片文件。

响应包含图片 ID、原图地址、风险等级、摘要和隐私检测项。

## POST /api/mask

根据 `imageId`、`maskType` 和待处理区域对图片进行打码处理。

```json
{
  "imageId": "img_xxx",
  "maskType": "black",
  "items": [
    { "x": 120, "y": 250, "width": 260, "height": 50 }
  ]
}
```

`maskType` 支持 `black`、`blur`、`mosaic`。

## GET /api/history

返回最近图片隐私检测记录。

## POST /api/scam/analyze

Scam Radar 文本反诈接口。

```json
{
  "text": "奖学金补贴今日截止，请点击链接填写银行卡和验证码。"
}
```

响应包含风险等级、规则评分、命中原因和建议。

## POST /api/link/check

Link Guard 链接风险检测接口。

```json
{
  "url": "http://bit.ly/scholarship-verify"
}
```

响应包含规范化 URL、检查项和建议。

## POST /api/doc/check

Doc Shield 提交护盾接口。用户输入提交要求，并上传一个或多个材料文件。系统会解析提交要求，检查材料完整性、文件格式与命名规范，并识别文档文本中的隐私风险。

请求类型：`multipart/form-data`

字段：

- `requirement_text`：用户输入的提交要求文本。
- `files`：一个或多个材料文件。

当前支持内容读取：

- `txt`
- `md`
- `pdf`
- `docx`

`png`、`jpg`、`zip` 暂不解析内容，只检查文件名、后缀和上传状态。

响应示例：

```json
{
  "riskLevel": "high",
  "score": 72,
  "summary": "发现 1 项高优先级问题，建议修正后再提交。",
  "parsedRequirements": {
    "formats": ["pdf"],
    "namingRule": "学号-姓名-课程论文",
    "requiredMaterials": ["封面", "摘要", "正文", "参考文献"],
    "lengthRequirement": "不少于3000字",
    "deadline": "2026年7月20日 18:00 前",
    "rawText": "请于 2026年7月20日 18:00 前提交 PDF 文件..."
  },
  "files": [
    {
      "fileName": "20240001-张三-课程论文.pdf",
      "extension": "pdf",
      "contentType": "application/pdf",
      "size": 235000,
      "status": "parsed",
      "wordCount": 3250,
      "pageCount": 8
    }
  ],
  "checks": [
    {
      "category": "format",
      "label": "已上传 pdf 格式材料",
      "evidence": "20240001-张三-课程论文.pdf",
      "riskLevel": "low",
      "status": "pass"
    },
    {
      "category": "privacy",
      "label": "手机号风险",
      "evidence": "20240001-张三-课程论文.pdf：联系电话 13812345678",
      "riskLevel": "high",
      "status": "fail"
    }
  ],
  "suggestions": [
    "提交前删除或遮盖无关手机号、身份证号、银行卡号、家庭住址、紧急联系人等个人敏感信息。"
  ]
}
```

## Doc Shield 使用示例

输入课程论文提交要求：

```text
请于 2026年7月20日 18:00 前提交 PDF 文件，命名规则为 学号-姓名-课程论文。材料需包含封面、摘要、正文、参考文献；正文不少于3000字。
```

上传 `20240001-张三-课程论文.pdf` 后，系统会检查：

- 命名是否大致符合“学号-姓名-课程论文”。
- 文件格式是否符合 PDF 要求。
- 内容是否包含封面、摘要、正文、参考文献。
- 文本中是否存在手机号、身份证号、银行卡号、地址等隐私风险。
- 生成总体风险等级、提交安全评分和修改建议列表。

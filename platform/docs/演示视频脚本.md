# GuardianHub 演示视频脚本

目标时长：5 分钟以内

## 0:00-0:40 项目背景

大家好，我们的项目是 GuardianHub：面向高校场景的 AI 数字安全防护平台。它由原 Privacy Sentinel 隐私哨兵升级而来，面向高校学生在图片分享、聊天沟通、链接访问、材料提交中的数字安全风险。

## 0:40-1:20 核心思路

项目的核心不是做单点工具，而是把风险检查前置到关键操作之前。GuardianHub 围绕分享前、转账前、点击前、提交前四个节点，提供 Privacy Sentinel 隐私哨兵、Scam Radar 反诈雷达、Link Guard 链接卫士和 Doc Shield 提交护盾四个模块。

## 1:20-3:20 功能演示

打开 PC Web Demo。首页展示 GuardianHub 安全中心，包括四个模块、今日安全评分、平台介绍和最近检测记录。

首先进入 Privacy Sentinel 隐私哨兵，上传示例图片，点击“开始隐私检测”。系统返回检测结果，在图片上标出手机号、地址、二维码、订单号和昵称等区域。右侧风险概览展示整体风险等级、检测项数量和 high / medium / low 风险统计。接着选择黑条、模糊或马赛克，一键处理高风险信息，后端会根据检测框对原图进行真实处理，并在安全预览区域展示处理后的图片。

然后进入 Scam Radar 反诈雷达，粘贴聊天文本，系统识别奖学金补贴、验证码、银行卡、紧迫话术等风险并给出建议。进入 Link Guard 链接卫士，输入 URL，系统检查 HTTPS、短链接、可疑关键词和异常域名。最后进入 Doc Shield 提交护盾，输入材料文件名，展示命名、隐私字段和材料清单检查结果。

## 3:20-4:10 技术说明

当前阶段是初赛验证版，采用 React + Vite + TypeScript 构建前端，FastAPI 提供接口，Pillow 完成真实图像打码。检测结果暂时使用 mock 数据，避免过早引入复杂 OCR 或付费大模型 API，优先保证本地 Demo 稳定运行。

## 4:10-4:40 创新点

GuardianHub 的创新点在于从单一隐私工具升级为高校数字安全平台，用统一的 high / medium / low 风险等级、命中原因和建议操作，把分散风险整合到一个安全中心。

## 4:40-5:00 总结与规划

后续我们会接入 OCR、二维码识别、人脸检测、文件解析和更完善的风险识别模型，并迁移到 HarmonyOS / ArkTS 多端体验。手机端负责快速入口，平板端负责精细处理，手表端负责高风险提醒。

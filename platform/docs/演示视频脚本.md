# GuardianHub 演示视频脚本

目标时长：5 分钟以内

## 0:00-0:40 项目背景

大家好，我们的项目是 GuardianHub：面向高校场景的 AI 数字安全防护平台。它由原 Privacy Sentinel 隐私哨兵升级而来，面向高校学生在图片分享、代码提交、链接访问、材料提交中的数字安全风险。

## 0:40-1:20 核心思路

项目核心不是做单点工具，而是把风险检查前置到关键操作之前。GuardianHub 围绕分享前、提交前、点击前、上传前四个节点，提供 Privacy Sentinel 隐私哨兵、Code Guardian 代码卫士、Link Guard 链接卫士和 Doc Shield 提交护盾四个模块。

## 1:20-3:30 功能演示

打开 PC Web Demo，首页展示 GuardianHub 安全中心，包括四个模块、今日安全评分、平台介绍和最近检测记录。

首先进入 Privacy Sentinel 隐私哨兵，上传示例图片，系统返回检测结果，在图片上标出手机号、地址、二维码、订单号和昵称等区域。选择黑条、模糊或马赛克后，后端会根据检测框对原图进行真实处理，并展示处理后的图片。

然后进入 Code Guardian 代码卫士。粘贴一段包含硬编码 API Key、SQL 拼接、命令执行和敏感日志的示例代码，点击“开始代码安全检测”。系统输出风险等级、安全评分、命中行号、代码片段、风险原因、修改建议，以及是否建议直接提交代码。

接着进入 Link Guard 链接卫士。输入包含 login、redirect、token 的 URL，并选择来源为短信。系统从 HTTPS、可疑参数、关键词、随机 token 和来源场景生成链接安全体检报告，展示是否建议打开链接。二维码上传区域当前作为占位，提示用户手动输入二维码解析出的链接。

最后进入 Doc Shield 提交护盾。输入课程论文提交要求并上传材料，系统生成提交检查报告，展示格式、命名规则、必需材料、隐私风险和提交建议。

## 3:30-4:15 技术说明

当前阶段是初赛验证版，采用 React + Vite + TypeScript 构建前端，FastAPI 提供接口，Pillow 完成真实图像打码。Code Guardian 和 Link Guard 均使用本地规则、正则和关键词实现，不接入大模型 API，也不引入复杂依赖，优先保证本地 Demo 稳定运行。

## 4:15-4:45 创新点

GuardianHub 的创新点在于从单一隐私工具升级为高校数字安全平台，用统一的 high / medium / low 风险等级、0-100 安全评分、命中证据和建议操作，把分散风险整合到一个安全中心。

## 4:45-5:00 总结与规划

后续我们会逐步接入 OCR、二维码识别、代码项目级扫描、依赖漏洞库、校内可信通知校验和 HarmonyOS / ArkTS 多端体验。当前版本先把关键流程跑通，让每个模块都能稳定演示、可解释输出。

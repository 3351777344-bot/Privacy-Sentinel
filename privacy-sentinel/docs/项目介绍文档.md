# Privacy Sentinel 隐私哨兵项目介绍

## 项目背景

截图、快递单、订单页、聊天记录和证件照片已经成为日常分享中最常见的图片内容。用户在转发图片时，经常会忽略手机号、地址、订单号、二维码、头像昵称等隐私信息，导致个人身份、交易记录、位置轨迹甚至账号入口被无意公开。

## 用户痛点

传统图片编辑工具需要用户主动发现隐私信息，再手动框选、涂抹和保存。这个过程依赖个人经验，容易漏掉二维码、订单号、聊天昵称等半隐性信息。对于校园用户、网购用户和社交平台用户来说，分享动作高频且快速，隐私检查却往往发生得太晚。

## 项目定位

Privacy Sentinel 是一款 AI 隐私安全分享助手。它把“分享前检查”作为核心入口，在用户准备分享图片前主动检测风险，标注敏感区域，给出风险等级，并一键生成安全分享版图片。

## 核心功能

- 图片上传与预览：支持截图、聊天记录、订单页、快递单等图片。
- 隐私风险检测：识别手机号、地址、二维码、订单号、昵称等敏感信息。
- 风险分级：将检测项划分为高风险、中风险、低风险。
- 敏感区域标注：在图片上直观显示隐私区域和风险等级。
- 一键打码：支持黑条、模糊、马赛克三种处理方式。
- 安全预览：展示处理后的图片并给出分享建议。
- 历史记录：沉淀最近检测和处理状态，形成安全闭环。

## 技术路线

当前阶段采用 React + Vite + TypeScript 构建 PC Web Demo，FastAPI 提供后端接口，Pillow 完成真实图像打码。检测逻辑先使用 mock 数据，保证本地演示稳定可靠。后续可逐步接入 OCR、二维码识别、人脸检测和更完善的风险分类模型。

## 创新点

Privacy Sentinel 不把自己定位为普通图片编辑器，而是面向“分享前隐私风险检测”的 AI 安全助手。它将主动检测、风险分级、区域标注、一键打码和安全预览串成闭环，降低用户发现和处理隐私信息的门槛。

## 应用前景

项目可应用于社交分享、电商售后、校园资料流转、办公协作、社区发布等场景。尤其在 HarmonyOS 多端环境中，可进一步形成手机快速检测、平板精修处理、手表高风险提醒的协同体验。

## 后续 HarmonyOS 落地方案

复赛阶段计划将当前 PC Web Demo 迁移为 HarmonyOS ArkTS 前端。手机端重点承接分享入口和快速检测，平板端提供更大的图片编辑与批量处理空间，手表端用于关键风险提醒。后续再完成 HAP 打包、端云接口接入和多端协同体验优化。
